# AIコーディング実践ガイド  
（VS Code / GitHub / Python・PowerShell / uv / Git / Copilot）

## ✅ このドキュメントの目的
生成AIの発達により、**「コードを書くスキル」より「コードを理解し、AIに意図を伝える力」**が重要になっています。

本ガイドでは、非エンジニアでも AI を活用して業務に必要なスクリプト開発ができるよう、

- 現場で本当に使われているツール選定理由
- AIと相性の良い言語やワークフロー

## ✅ 1. VS Code（Visual Studio Code）とは？

### ● IDEとは？
IDE（統合開発環境）は、プログラム作成に必要な要素がまとまった “作業場” です。

| IDEが提供するもの | 例 |
|------------------|----|
| テキスト編集 | コードを書ける |
| デバッグ | エラー箇所がわかる |
| 実行環境 | プログラムを動かせる |
| 拡張機能 | 言語・AI・Gitなど追加 |

### ● なぜ VS Code が最適か（重要ポイント）
| 理由 | 詳細 |
|------|------|
| **無料** | 個人・会社・教育用途すべてOK |
| **世界で最も使われているエディタ** | 解決方法やノウハウがネットに大量 |
| **拡張機能の質が高い** | Python, PowerShell, Docker, Remote開発など |
| **GitHubとの連携が強い** | Git操作、Copilot利用がボタン1つ |
| **軽量で高速** | スペックが低くても動く |
| **クロスプラットフォーム** | Windows / Mac / Linux共通の操作 |

✅ 特に **「非エンジニア × AI」** の現場では  
VS Code + Copilot が世界標準になりつつあります。

## ✅ 2. GitHubとは？

GitHubは **ソースコードの保管庫 + バージョン管理 + 協力作業ツール + AIサービス基盤**。

### ● GitHubを使う意味
| 利点 | 説明 |
|------|------|
| バージョン管理 | 過去に戻せる / 破壊的変更に強い |
| 共有が簡単 | URL渡すだけ。メール添付不要 |
| 自動バックアップ | PCが壊れても復元可 |
| セキュア | 企業でも利用実績が多い |
| Copilotが使える | GitHubアカウントで連携 |

✅ 「保管」「共有」「安全」「AI」の4つが一度に揃うため、企業利用が急増しています。

## ✅ 3. Python と PowerShell（なぜ AI と相性が良い？）

### ● Pythonの特徴
- 文法がシンプル → 初学者でも読める
- AI / Excel自動化 / Web / データ分析すべてに使える
- ライブラリが豊富で、AIが「書きやすい・直しやすい」
- コンパイル不要 → 保存して実行するだけ

### ● PowerShellの特徴
- **Windowsに標準搭載 → 導入不要**
- Excel、CSV、フォルダ操作、自動化で強い
- IT部門・現場双方で使える現実的言語
- 会社内の運用で最も「仕事を減らせる」

### ● どちらが適している？
| 用途 | 推奨言語 |
|------|----------|
| Excel処理・フォルダ整理・ファイル自動化 | PowerShell |
| データ分析・AI・業務ロジック多め | Python |

✅ どちらも **AIで書かせると精度が高い** → 独学より圧倒的に楽。

## ✅ 4. 仮想環境と uv

### ● 仮想環境が必要な理由（実務で重要）
1台のPCで複数のプロジェクトを扱うと、

- ライブラリのバージョンが衝突
- あるプロジェクトを更新したら別のが壊れる
- 動いていたプログラムが急に動かなくなる

など、現場で頻発します。

✅ 仮想環境は「プロジェクト専用の小さな独立したPython」を作り、  
混乱を完全に防ぐための仕組みです。

---

### ● なぜ uv が優れている？
| 項目 | uv | 従来 (pip/venv) |
|------|----|-----------------|
| 速度 | 爆速 | 遅い |
| 依存解決 | 自動・高精度 | 失敗しやすい |
| コマンドがシンプル | ✅ | ❌ |
| 企業利用増加中 | ✅ | - |

uvは「pipの完全上位互換 + 速い + 便利」。  
Python界隈では2023〜2025で急速に標準化が進んでいます。

---

### ● uv の基本コード

```bash
# プロジェクト作成（フォルダ＆環境構築）
uv init myproject
cd myproject

# パッケージ追加
uv add requests openpyxl

# インストールの同期
uv sync

# 仮想環境の有効化（Windows）
./.venv/Scripts/activate
✅ uv init → uv add → uv sync → activate
これだけ覚えれば現場で困らない。

✅ 5. AIコーディングとは？
「AIがコードを書く」のではなく、
人間が目的を伝え、AIがそれを実装し、人間が検証する開発方式。

例：

わたし → 「Excelを読み取り集計し、結果をCSVで出して。」

AI → コード生成

実行

必要なら修正を依頼

ドキュメントやテストも生成可能

● メリット
コーディング経験ゼロでも開発できる

作業時間を大幅短縮

エラー原因もAIに説明させられる

ドキュメント化が容易

● 失敗例（ありがち）
曖昧な指示

目的が説明されていない

入力・出力・例外を伝えていない

✅ AIに「人間が設計した意図」を伝えることが最重要

✅ 6. AIコーディングの進め方（成功率を上げる型）
AIに丸投げするのではなく
V字工程を小さく繰り返すと成功率が最大になる。

工程	AIへの依頼テンプレ例
① 要求定義	「Excelを読み込み、A列でグループ化し売上合計を出したい」
② 要件定義	「入力はこのExcel、出力はCSV。エラー処理とログも」
③ 設計	「処理の流れをフローチャートで説明して」
④ コーディング	「Pythonでコードを書いて。uv環境前提で」
⑤ リファクタリング	「可読性を上げて。関数に分割して」
⑥ テスト	「pytest用ユニットテストを生成して」
⑦ ドキュメント	「README.mdを作成して」

✅ 現場で一番大切なのは「丁寧な説明 × 小さく検証」。

✅ 7. Git（最低限必要な操作だけ）
● Gitは何を解決する？
上書きでファイルが壊れる問題

メール添付でバージョンが行方不明

人によって違うフォルダに保存

● 基本コマンド（これだけ覚えれば実務で戦える）
bash
コードをコピーする
# リポジトリ取得
git clone <URL>

# 変更の記録
git add .
git commit -m "変更内容を短く書く"
git push    # GitHubへ送る

# 最新を取得
git pull

# ブランチ作業
git checkout -b 修正A
git checkout main
✅ 開発で壊しても git reset や git revert で戻せる
→ だから怖がらなくて良い。
